MT-Congés

Application Java de gestion des congés développée pendant mon alternance chez MTB111

Catégorie Application Java
Client MTB111
Durée 6 mois
Rôle Développeur principal
Année 2024

Présentation du projet

Dans le cadre de mon alternance chez MTB111, j'ai été chargé de développer une application dédiée à la gestion des congés pour l'entreprise. Ce projet répond à un besoin concret d'optimisation des processus RH internes.

MT-Congés est une application complète permettant aux employés de soumettre leurs demandes de congés, aux responsables de les valider, et au service RH de disposer d'une vue d'ensemble des absences au sein de l'entreprise.

Objectifs

L'application a été conçue pour répondre à plusieurs objectifs spécifiques :

  • Simplifier le processus de demande de congés pour les employés
  • Faciliter la validation des demandes par la hiérarchie
  • Offrir une vue d'ensemble des absences pour faciliter la planification des ressources
  • Automatiser les calculs des jours de congés restants
  • Générer des rapports et statistiques pour le service RH

Fonctionnalités

Pour les employés

  • Interface intuitive pour soumettre des demandes de congés
  • Visualisation du solde de congés restant
  • Historique des demandes précédentes
  • Notifications en temps réel sur l'état des demandes

Pour les responsables

  • Tableau de bord des demandes en attente
  • Système de validation avec commentaires
  • Vue d'ensemble des congés de l'équipe
  • Outil de planification des ressources

Pour le service RH

  • Administration des utilisateurs et des droits
  • Configuration des types de congés
  • Génération de rapports statistiques
  • Export des données pour la paie

Développement technique

J'ai développé cette application en utilisant les technologies suivantes :

  • Backend : Java avec Spring Boot pour la logique métier et l'API REST
  • Persistance : MySQL avec Hibernate ORM pour la gestion des données
  • Frontend : JavaFX pour l'interface utilisateur desktop
  • Sécurité : Spring Security pour l'authentification et les autorisations
  • Tests : JUnit et Mockito pour les tests unitaires et d'intégration

L'architecture de l'application suit le modèle MVC (Modèle-Vue-Contrôleur) avec une séparation claire des responsabilités :

  • Les entités et repositories pour la gestion des données
  • Les services pour la logique métier
  • Les contrôleurs pour la gestion des requêtes
  • Les vues JavaFX pour l'interface utilisateur

Résultats et bénéfices

L'implémentation de MT-Congés a apporté plusieurs avantages significatifs à l'entreprise :

  • Réduction de 75% du temps de traitement des demandes de congés
  • Diminution des erreurs administratives liées à la gestion manuelle
  • Amélioration de la visibilité des absences pour une meilleure planification
  • Satisfaction accrue des employés grâce à un processus plus transparent
  • Données fiables pour la prise de décision RH

Le projet a été très bien accueilli par les utilisateurs et est désormais utilisé quotidiennement par l'ensemble des collaborateurs de MTB111.

Défis et apprentissages

Ce projet m'a confronté à plusieurs défis techniques et organisationnels :

  • Conception d'une interface utilisateur intuitive adaptée à différents profils d'utilisateurs
  • Mise en place d'un système de notification en temps réel
  • Gestion des calculs complexes de jours de congés (jours fériés, temps partiel, etc.)
  • Développement d'un système de rapport flexible et performant

J'ai beaucoup appris durant ce projet, notamment sur :

  • Le développement d'applications d'entreprise robustes
  • La gestion de projet et la communication avec les parties prenantes
  • L'importance des tests et de la documentation
  • L'optimisation des performances d'une application Java
